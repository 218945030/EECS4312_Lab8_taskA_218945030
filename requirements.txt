## 1. Problem Understanding - Describe the problem your system solves.
System purpose: Automated meeting scheduling system
Inputs to the system: Working Hours, Busy Intervals, Duration, Buffer
Outputs produced by the system: Meeting Times
Primary goal of the system: The objective of the systems is to help teams find times to schedule their meetings within their specified work hours. 
---
## 2. Assumptions - List assumptions about the environment, users, or
inputs.
A1: System will be used in a professional enviorment 
A2: System users will be team leads 
A3: System inputs will be intervals of time 
A4: System will be automated 
A5: Team leads will rely on system consitency 
A6: System will be easy to use without training 
---
## 3. Functional Requirements - Describe the main behaviors the system
must perform.
FR1: The system must accept the interval of working hours. 
FR2: The system must accept a list of busy intervals where meetings cannot be scheduled. 
FR3: The system must accept a duration of the meeting to be scheduled.  
FR4: The system must accept a buffer time that must pass between two meetings. 
FR5: The system must return a list of possible meeting times, that fall within the given constraints, in a chronological order.  
FR6: The system must only require user interaction for inputs.
FR7: The system must automatically handle conflicts 
FR8: The system must provide clear explainations for scheduling decisions 
FR9: The system must explicitly handle edge cases

## 5. Acceptance Criteria - Acceptance criteria define how system
behavior will be verified.
AC1:
Given a suggested meeting time 
When it is within the specified working hours
Then it is a valid input 
Linked Constraint ID(s): C#1
AC2:
Given a suggested meeting time
When it overlaps with a defined busy interval
Then it is not a valid meeting time to suggest. 
Linked Constraint ID(s): C#1
AC3:
Given The start and end times of the working hours interval 
When the start time is greater than 0:00 and the end time is less than 24:00 
Then the working hours interval is valid.
Linked Constraint ID(s): C#7
AC4:
Given a busy interval
When the iterval is within the specified working hours
Then the busy interval is valid
Linked Constraint ID(s): C#1
AC5:
Given a list suggested meeting times
When the list is outputed
Then the meeting times must be listed in chronological order. 
Linked Constraint ID(s): C#6
AC6:
Given A suggested meeting time 
When the suggested meeting time overlaps with specified buffer after another meeting or busy interval
Then the suggested meeting time is not valid. 
Linked Constraint ID(s): C#1
AC7:
Given A interval of working hours, a list of busy intervals, and a buffer interval
When all of the given inputs are valid
Then the system will output all possible meeting times without any user input 
Linked Constraint ID(s): C#3
